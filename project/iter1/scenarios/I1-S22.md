# I1-S22 `cwf:setup --codex-wrapper` opt-in/decline

## 목적

wrapper opt-in(활성화)과 decline/rollback(비활성화) 경로를 검증한다.

## 실행

- 로그: [project/iter1/artifacts/I1-S22.log](../artifacts/I1-S22.log)
- 스크립트:
  - `install-wrapper.sh --enable --scope project --project-root ...`
  - `install-wrapper.sh --disable --scope project --project-root ...`
  - `install-wrapper.sh --status`

## 관찰

- enable 후 `Active: true`, wrapper link 생성 확인
- disable 후 `Active: false`, wrapper symlink 제거 확인
- system `codex` 바이너리는 별도로 계속 존재
